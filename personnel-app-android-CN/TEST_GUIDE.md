# 测试指南

## 测试环境

### 推荐设备
- **真机**: 华为/小米/OPPO/vivo 等主流品牌
- **模拟器**: Android Studio Emulator (API 24+)
- **最低配置**: 2GB RAM, 1280x720 分辨率

## 测试步骤

### 1. 安装应用

```bash
# 方式 1: Android Studio 直接运行
# 点击 Run 按钮

# 方式 2: 命令行安装
adb install app/build/outputs/apk/debug/app-debug.apk

# 方式 3: 手动安装
# 将 APK 传输到设备，点击安装
```

### 2. 权限授予

首次启动时，应用会请求以下权限：
- ✅ 存储权限（读取/写入文件）
- ✅ 网络权限（数据同步，可选）

点击"允许"授予权限。

### 3. 功能测试

#### 测试用例 1: 查看人员列表
1. 启动应用
2. 检查是否显示人员列表
3. 验证列表中显示：姓名、职务、电话
4. 检查驻村领导标识

**预期结果**: 列表正常显示，数据正确

#### 测试用例 2: 搜索功能
1. 在搜索框输入"张"
2. 验证显示所有姓张的人员
3. 输入拼音"zhangsan"
4. 验证能搜索到"张三"
5. 输入首字母"zs"
6. 验证能搜索到"张三"

**预期结果**: 中文/拼音/首字母搜索均有效

#### 测试用例 3: 人员类型筛选
1. 点击"镇府干部" Chip
2. 验证只显示镇府干部
3. 点击"全部" Chip
4. 验证显示所有人员

**预期结果**: 筛选功能正常

#### 测试用例 4: 查看详情
1. 点击列表中任意人员
2. 验证跳转到详情页面
3. 检查基本信息、职务信息、履历等
4. 点击返回按钮

**预期结果**: 详情页数据完整，返回正常

#### 测试用例 5: 新增人员
1. 点击悬浮添加按钮（+）
2. 填写必填字段（姓名、电话、身份证）
3. 点击保存
4. 验证返回列表，新人员在列表中

**预期结果**: 新增成功，数据保存

#### 测试用例 6: 编辑人员
1. 在详情页点击编辑按钮
2. 修改任意字段
3. 点击保存
4. 验证修改生效

**预期结果**: 修改成功，数据更新

#### 测试用例 7: 删除人员
1. 在详情页点击菜单 -> 删除
2. 确认删除
3. 验证列表中该人员消失

**预期结果**: 删除成功

#### 测试用例 8: 导出数据
1. 点击菜单 -> 导出
2. 选择保存位置
3. 验证文件生成
4. 用 Excel 打开检查数据

**预期结果**: Excel 文件正常，数据完整

#### 测试用例 9: 导入数据
1. 准备测试 Excel 文件
2. 点击菜单 -> 导入
3. 选择文件
4. 验证数据导入成功

**预期结果**: 导入成功，数据正确

#### 测试用例 10: 数据加密
1. 新增人员，填写身份证号和电话
2. 导出 CSV 文件
3. 用文本编辑器打开
4. 验证身份证号和电话为加密状态（Base64）

**预期结果**: 敏感数据已加密

## 性能测试

### 启动时间
- **冷启动**: < 2 秒
- **热启动**: < 1 秒

### 列表滚动
- 1000 条数据滚动流畅，无卡顿

### 搜索响应
- 输入后即时响应，延迟 < 100ms

## 已知问题

1. **Android 13+ 权限**: 需要单独请求媒体权限
2. **大文件导出**: 超过 10000 条数据时导出较慢
3. **加密兼容性**: Android 7.0 以下 KeyStore 不可用，使用降级方案
